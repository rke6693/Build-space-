Built for the four things that actually move the P&L

Every feature is one a CFO can see on a dashboard. Nothing that sounds good in a blog post and doesn't move dollars.

Smart routing

Pick the best provider per request, with automatic failover when one goes down.

Semantic cache

pgvector-backed cache matches near-duplicate prompts — not just exact hits.

Shadow evaluation

Live A/B against a cheaper candidate. Auto-downshift only when a judge agrees.

Budget guardrails

Per-key monthly limits, hard-block or warn. Stop surprise bills at the gateway.

About Keel

An independent project. We ship fast and write about the work.

How we work

Keel is run async and written-first. There are no live demos, no sales calls, and no quotes. The roadmap, changelog, and monthly metrics are all public. If you'd like to use Keel, clone it or sign up for the cloud — both paths are self-serve.

Get in touch

One channel: hello@keel.dev. We answer every email, usually within two business days. Bugs and feature requests live on GitHub Issues so the conversation is searchable.

What we don't do

No phone support, no custom enterprise contracts, no dedicated account managers. The Business tier is the ceiling of what we sell. If you need a human in the loop, Keel probably isn't for you — and that's fine.
